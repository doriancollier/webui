### Bug 1: PATCH goes to wrong session ID after navigation + remap — **Bug**
**Observed:** After the session remap (b314a190 → 57fc8adb), clicking model selector (Haiku 4.5) and permission mode (Accept Edits) briefly showed optimistic state then reverted to Sonnet/Default. Network analysis shows BOTH PATCH requests went to `27200842` (the session that was in the URL when the page first loaded this tab), not `57fc8adb` (the current active session). Session `57fc8adb` received zero GET requests for session metadata — `session.model` was always undefined, so model fell back to DEFAULT Sonnet.

**Expected:** PATCH goes to the session currently shown in the URL (`57fc8adb`).

**Root cause:** `sessionId` in `updateSession` (via `useCallback` closure in `use-session-status.ts`) is stale — it references the session ID from the previous page load (`27200842`) rather than the current session. The URL correctly shows `?session=57fc8adb` but the hook's `sessionId` hasn't updated. This likely means `sessionId` is not being passed correctly through the component tree after a navigation/remap cycle — the `useSessionId()` hook or the prop chain hasn't propagated the new value.

**Evidence:**
- Network: 2 PATCHes to `27200842` (both 200), 0 PATCHes to `57fc8adb`
- Network: 0 GET requests for `57fc8adb` session metadata (only `/messages` 503s)
- Server logs: Only 1 PATCH logged (my manual curl at 15:45:37), no UI-originated PATCH

### Bug 2: Model/permission selectors are no-ops before first message — **UX Issue**
**Observed:** Before sending any message (no `?session=` in URL, `sessionId = null`), clicking Haiku in model dropdown and Accept Edits in permission dropdown had no effect. No visual feedback. Status bar continued showing default values.

**Expected:** Either (a) the selection is queued and applied on first session creation, or (b) the control is visually disabled/grayed out with a tooltip explaining why.

**Root cause:** `updateSession` in `use-session-status.ts:93` has `if (!sessionId) return;` — silently drops the change when no session exists. The UI renders the selectors as fully interactive with no indication they won't work.
