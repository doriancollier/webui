# Upgrade electron-vite from v3.1.0 to v5.0.0 (+ Electron ecosystem)

**Slug:** upgrade-electron-vite
**Author:** Claude Code
**Date:** 2026-04-01
**Branch:** preflight/upgrade-electron-vite

## 1) Intent & Assumptions

- **Task brief:** The desktop app (`apps/desktop`) is on electron-vite 3.1.0 (released March 2025). The latest stable is v5.0.0 (released December 2025). Upgrade electron-vite to v5 **and** upgrade Electron itself plus any other Electron ecosystem packages that naturally belong in the same PR. All upgrades land in one PR.
- **Packages in scope:**
  - `electron-vite`: `^3.1.0` ‚Üí `^5.0.0`
  - `electron`: `^33.4.11` ‚Üí latest compatible (see Decision 2)
  - `electron-builder`: `^26.8.1` ‚Üí latest
  - `electron-updater`: `^6.8.3` ‚Üí latest
  - `@electron/rebuild`: `^3.7.2` ‚Üí latest
  - `electron-log`: `^5.4.3` ‚Üí latest
  - `@vitejs/plugin-react`: `5.1.4` ‚Üí latest Vite 7-compatible
  - `@tailwindcss/vite`: `^4.1.18` ‚Üí latest
- **Assumptions:**
  - The upgrade is expected to be a low-risk, low-diff change based on the codebase's current structure
  - Both local (Node 22.17.1) and CI (`node-version: 22` ‚Üí latest 22.x) satisfy the new Node ‚â•22.12 minimum
  - **Electron version target:** Electron 39.x ‚Äî last version with Node.js 22.x, fully validated by electron-vite v5. Electron 40/41 ships Node.js 24, which electron-vite v5 has no build target for
- **Out of scope:**
  - Adding new electron-vite v5 features (e.g., `build.isolatedEntries`)
  - Setting up code signing or notarization (already deferred)
  - Upgrading Electron-unrelated monorepo dependencies

## 5) Research

### electron-vite Breaking Changes: v3.1.0 ‚Üí v4.0.0 (July 2025)

| Change                                     | Impact on this project                                                          |
| ------------------------------------------ | ------------------------------------------------------------------------------- |
| Node.js minimum bumped to ‚â•20.19 or ‚â•22.12 | ‚úÖ No action ‚Äî local is 22.17.1, CI is `node-version: 22` (latest 22.x)         |
| Vite v7 adopted (Vite 6 ‚Üí 7)               | ‚úÖ No action ‚Äî bundled in electron-vite; renderer plugins are Vite 7-compatible |
| CJS build of electron-vite package removed | ‚úÖ No action ‚Äî config already uses ESM imports                                  |
| Electron 36‚Äì37 support added               | ‚úÖ No impact                                                                    |

### electron-vite Breaking Changes: v4.0.0 ‚Üí v5.0.0 (December 2025)

| Change                                                                    | Impact on this project                                    |
| ------------------------------------------------------------------------- | --------------------------------------------------------- |
| `externalizeDepsPlugin` deprecated ‚Üí `build.externalizeDeps` (default on) | **Action required** ‚Äî remove plugin from main and preload |
| `bytecodePlugin` deprecated ‚Üí `build.bytecode` config option              | ‚úÖ No action ‚Äî not used                                   |
| Function-based nested config removed                                      | ‚úÖ No action ‚Äî project uses static config                 |
| Electron 18‚Äì21 support dropped                                            | ‚úÖ No action ‚Äî upgrading to ‚â•39                           |
| `build.isolatedEntries` added                                             | Optional future enhancement                               |

### Electron Version Compatibility

electron-vite v5.0.0 explicitly validates Electron 32‚Äì39 (added Electron 39 target in 5.0.0 release). Electron's latest stable as of Apr 2026 is **41.1.1**. Options:

| Target        | Status                                                   | Risk                                                                                                                         |
| ------------- | -------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| Electron 39.x | Officially supported by electron-vite v5                 | Low                                                                                                                          |
| Electron 41.x | Latest stable; not explicitly listed in electron-vite v5 | Low-medium ‚Äî build target may be slightly sub-optimal but app should run correctly since Electron 39‚Äì41 all use Node.js 22.x |

### Config Change Required (electron-vite)

**Before (v3 style):**

```typescript
import { defineConfig, externalizeDepsPlugin } from 'electron-vite';

export default defineConfig({
  main: {
    plugins: [externalizeDepsPlugin()],
    build: {
      outDir: 'dist/main',
      rollupOptions: {
        external: ['better-sqlite3', '@anthropic-ai/claude-agent-sdk'],
      },
    },
  },
  preload: {
    plugins: [externalizeDepsPlugin()],
    build: { outDir: 'dist/preload' },
  },
  // ...
});
```

**After (v5 style):**

```typescript
import { defineConfig } from 'electron-vite';

export default defineConfig({
  main: {
    // externalizeDeps is on by default in v5 ‚Äî plugin not needed
    build: {
      outDir: 'dist/main',
      rollupOptions: {
        external: ['better-sqlite3', '@anthropic-ai/claude-agent-sdk'],
      },
    },
  },
  preload: {
    // externalizeDeps is on by default in v5 ‚Äî plugin not needed
    build: { outDir: 'dist/preload' },
  },
  // ...
});
```

Note: `rollupOptions.external` entries (`better-sqlite3`, `@anthropic-ai/claude-agent-sdk`) are Rollup-level externals independent of `externalizeDepsPlugin`. They must stay.

### Recommendation

**Direct upgrade of all in-scope packages in a single PR.** The electron-vite migration delta is tiny (~3 lines). The Electron and ecosystem package bumps are routine version upgrades with no expected source code changes. The main risk is Electron API breakage from 33‚Üí39/41 ‚Äî the spec phase should check Electron release notes for any deprecated APIs used in `main/`, `preload/`, and `window-manager.ts`.
